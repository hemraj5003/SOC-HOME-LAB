# SOC-HOME-LAB
Introduction

I designed this SOC home lab to understand how security teams monitor systems, detect suspicious activities, and investigate security events using a SIEM (Security Information and Event Management) platform. The lab simulates attacker behavior and demonstrates how security logs are generated, collected, and analyzed to identify potential threats.

The environment allows testing of basic attack techniques such as network scanning using Nmap, SMB share enumeration, and simple DoS simulations using hping3. These activities help observe the behavior of the Windows system under different attack scenarios and analyze the security logs generated during these events.

Through this home lab, I explored both offensive and defensive security workflows. From the offensive perspective, I simulated reconnaissance and enumeration activities to understand how attackers begin exploring a target system and expand their attack surface. From the defensive perspective, I monitored the logs and alerts generated by the system using Wazuh SIEM, which helped me understand how security teams detect and investigate suspicious activities.

Wazuh Installation

Running Wazuh requires a system with sufficient resources. For this lab environment, a minimum of 6 GB RAM is recommended to ensure stable performance of the Wazuh manager and dashboard.

The Wazuh manager was installed on an Ubuntu server using the official installation script.

Download the installation script:

curl -sO https://packages.wazuh.com/4.14/wazuh-install.sh

Run the installation:

sudo bash wazuh-install.sh -a

After the installation process completed, the Wazuh dashboard became accessible through the browser, allowing monitoring and investigation of security events.

Project Motivation

I initially planned to create a SOC home lab because I wanted to understand how real security infrastructures work in large organizations. I was curious about how companies monitor their systems, detect attacks, and investigate security incidents.

At the beginning, I watched several tutorials and videos to get a quick understanding of SOC workflows and SIEM tools. While these resources helped explain the concepts, when I tried to build the lab myself I encountered multiple installation and configuration errors.

During this process, I realized that an important question had not been clearly answered:

“What should the architecture of my SOC home lab look like?”

To answer this, I began exploring how IT companies design their security infrastructure and monitoring architectures. I researched how different systems communicate with each other, how logs are collected, and how security monitoring platforms are deployed in real environments.

Based on this understanding, I built a basic SOC home lab architecture that allowed me to simulate attacks and observe how logs are generated and analyzed using Wazuh.

Although this setup is a basic implementation, I plan to continue expanding it by adding more attack simulations, monitoring components, and detection techniques to strengthen my understanding of security monitoring, threat detection, and SOC operations.
